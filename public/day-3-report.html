🔴 Issues by Priority

HIGH

Client Edit is Missing

There is no way to edit a client's name, email, phone, address, company, tags, or notes anywhere in the app. The clients list only offers "View Profile" and "Delete". The client detail page is read-only.

FixAdd an Edit dialog to the client list (similar to Staff edit) OR an edit form on the client detail page.
HIGH

Header Search Does Nothing

The search bar in the top header is the most prominent UI element on every page, but it has no functionality. Users type in it and nothing happens.

FixWire it to a search query across clients and appointments — either client-side filtering or a Supabase full-text search (ilike).
HIGH

Sidebar Shows Hardcoded User

The sidebar footer shows "Admin User / admin@nexus.com" with "AW" initials regardless of who is logged in. The real user profile is already fetched in the layout but only passed to the Header.

FixPass the user prop from layout.tsx to <Sidebar user={user} /> and render real name/email/initials.
HIGH

Sidebar Sign Out Doesn't Work

The LogOut button in the sidebar footer has no onClick handler. Users who click it expect to be signed out but nothing happens. This is a confusing UX break.

FixAdd onClick={handleSignOut} to the sidebar's LogOut button, using the same Supabase signOut pattern as the header.
MEDIUM

Notifications Can't Be Marked as Read

Clicking a notification in the bell dropdown never updates is_read in the database. The unread count badge (red dot) never goes away.

FixAdd an onClick to each notification row that calls supabase.from("notifications").update({ is_read: true }).eq("id", notif.id) and updates local state.
MEDIUM

Client Status Can't Be Changed

Once a client is created, their status (active / VIP / prospect / inactive) cannot be changed from the UI. There is no inline toggle or edit field for status.

FixAdd a status Select to the client edit dialog (once edit is built), or add a quick inline status dropdown on the client list row.
MEDIUM

Denormalized Counts Never Sync

staff.total_appointments, clients.total_appointments, and clients.total_revenue are set at seed time and never updated. Staff cards, client profiles, and the Reports "top staff/clients" rankings all show stale data.

FixEither add Supabase DB triggers on the appointments table to auto-update these columns, or update them manually in the create/delete appointment handlers.
MEDIUM

"Book Appointment" Doesn't Pre-fill Client

The "Book Appointment" button on the client detail page goes to /appointments — not /appointments?new=1&clientId=xxx. The dialog doesn't open and the client isn't pre-selected.

FixChange the href to /appointments?new=1&clientId={client.id} and read clientId from searchParams in the appointments page to pre-select it in the form.
LOW

Settings Page is UI-Only

All settings (business info, notification preferences, 2FA, billing plan) are fake. "Save Changes" shows a green flash but writes nothing to the database.

FixCreate a settings or workspace table in Supabase and wire Save to upsert(). At minimum, notification preferences could be stored in the profiles table.
LOW

Sidebar Badges Are Hardcoded

"128" (Clients) and "4" (Appointments) in the sidebar navigation are static strings — not live counts from the database.

FixFetch real counts in the dashboard layout and pass them to the Sidebar, or remove the badges and show them only on the respective pages.
LOW

Dark Mode Toggle is Decorative

The moon/sun icon switches but applies no styling. No dark theme is implemented.

FixUse document.documentElement.classList.toggle("dark") and add Tailwind dark: variants, or integrate next-themes.
LOW

"No-show" Status Unreachable

The "no-show" status exists in the DB enum and is rendered on the calendar, but there is no button to set it. The detail dialog only offers Confirm and Cancel.

FixAdd a "Mark No-show" button to the appointment detail dialog footer (alongside Confirm/Cancel), visible only when status is scheduled or confirmed.
LOW

"Add Note" Button Has No Handler

On the client detail → Notes tab, "Add Note" renders but clicking it does nothing. Client notes can only be added at creation time via /clients/new.

FixAdd a small textarea dialog that inserts a row into client_activities (type: "note") and refreshes the activity timeline.
