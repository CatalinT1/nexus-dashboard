SaaS Completeness
Honest assessment: The core domain logic (clients, staff, appointments, users) is well-built and live on Supabase. But the surrounding SaaS infrastructure — billing, notifications, search, settings persistence, audit logs, onboarding — is largely absent. These are the features that turn a working dashboard into a shippable product.

🗺️ Recommended Build Order

Ordered by what blocks a real product launch vs. what can wait. Features are grouped into four waves.

🔴 Wave 1 — Fix Before Going Live

CRITICAL

Multi-Tenancy

If this is a product sold to multiple clinics, every clinic's data is currently visible to every other clinic. Add organization_id to all tables and scope RLS policies to tenant.

  • New organizations table
  • organization_id FK on clients, staff, appointments
  • RLS updated to filter by org
Effort: Large — architectural change
CRITICAL

Sidebar — Real User + Logout

Sidebar shows hardcoded "Admin User" and its logout button does nothing. Two small fixes that affect every logged-in user every session.

  • Pass user prop from layout.tsx to Sidebar
  • Add handleSignOut onClick to sidebar LogOut button
Effort: 30 minutes
CRITICAL

Global Header Search

The search bar is the most visible UI element on every page. Currently non-functional. Must at least search clients by name/email.

  • Add onChange handler with debounce
  • Supabase ilike query on clients + appointments
  • Results dropdown or navigate to /clients?q=term
Effort: Medium — 2–3 hours
CRITICAL

Client Edit

There is currently no way to edit a client's name, email, phone, address, or status after creation. A core CRM feature that must exist.

  • Add Edit dialog to clients list (like staff edit)
  • OR add edit form to client detail page
  • supabase.from("clients").update().eq("id")
Effort: Medium — 2–3 hours
CRITICAL

Notifications — Mark as Read

Unread badge never clears. Clicking notifications does nothing. Users will be confused and ignore the bell entirely.

  • onClick on notification row → update is_read in DB
  • Decrement unreadCount in local state
  • "Mark all as read" button
Effort: Small — 1 hour
CRITICAL

Error Tracking (Sentry)

Production errors are currently invisible. A runtime crash in the appointments page would silently fail with no alert. 10-minute setup.

  • npm install @sentry/nextjs
  • DSN in .env, wrap in Sentry config
  • Errors auto-captured + alertable
Effort: Very small — 30 minutes

🟡 Wave 2 — Ship Within 2 Weeks

HIGH

Email Notifications

Send emails on key events: appointment created, confirmed, cancelled, staff assigned. Use Resend (easiest Supabase integration).

  • npm install resend
  • Supabase Edge Function triggered by DB insert
  • Templates: confirmation, reminder, cancellation
Effort: Medium-large — 4–6 hours
HIGH

Settings Page — Wire to Supabase

Save Changes currently does nothing. Business name, timezone, notification preferences should persist.

  • Create a settings or workspace table
  • Load on page mount, save on button click
  • Notification preferences stored in profiles table
Effort: Medium — 3–4 hours
HIGH

Denormalized Count Sync

staff.total_appointments and clients.total_revenue never update. Reports and rankings show stale seed data.

  • DB trigger on appointments INSERT/DELETE
  • OR update manually in create/delete handlers
  • Affects Reports "top staff" and "top clients"
Effort: Small — Supabase SQL trigger preferred
HIGH

CSV Export

No way to export data. Clinics need client lists, appointment history, and revenue reports in Excel/CSV.

  • Export button on clients list
  • Export on appointments page (filtered results)
  • Export on reports page
Effort: Small per page — 1–2 hours total
HIGH

"Book Appointment" Pre-fills Client

The button on the client detail page should open the create dialog with that client already selected.

  • Change href to /appointments?new=1&clientId=xxx
  • Read clientId from searchParams in appointments page
  • Pre-select client in createForm on open
Effort: Very small — 30 minutes
HIGH

Pagination

All lists load everything. A clinic with 500 clients would have an unusable, slow page.

  • Add .range(offset, offset+24) to Supabase queries
  • Next / Prev controls or infinite scroll
  • Affects clients, appointments, staff
Effort: Medium — 2–3 hours

🔵 Wave 3 — Build in Month 2

MEDIUM

Billing & Subscriptions

If this is a product sold to clinics, billing is required. Stripe + Supabase is the standard pattern.

  • Stripe Customer + Subscription per organization
  • Webhook: update plan in DB on payment events
  • Billing portal page (Stripe hosted)
Effort: Large — full feature
MEDIUM

Dark Mode

Toggle exists but does nothing. Implement via next-themes + Tailwind dark: variants.

  • npm install next-themes
  • Wrap layout in ThemeProvider
  • Add dark: prefix to key components
Effort: Medium — 2–4 hours
MEDIUM

Calendar Sync (Google / Outlook)

High value for clinic staff — sync their appointments to their personal calendar automatically.

  • Google Calendar API OAuth integration
  • Create event on appointment confirm
  • Update/delete on changes
Effort: Large — OAuth + API work
MEDIUM

Appointment Reminders (Scheduled)

24h and 1h before an appointment — send email/SMS reminder to client. Requires a job scheduler.

  • Supabase Edge Function with cron trigger
  • Query appointments starting within 24h / 1h
  • Send via Resend / Twilio
Effort: Medium — 3–4 hours
MEDIUM

Audit Log (System-wide)

Track who changed what and when — appointment status changes, role changes, client edits.

  • New audit_logs table: user_id, action, entity, before, after, timestamp
  • Insert row in each mutation handler
  • Admin view page
Effort: Medium — DB + handlers
MEDIUM

Social Login (Google)

Reduce friction for sign-in. Supabase supports Google OAuth with minimal config.

  • Enable Google provider in Supabase dashboard
  • Add "Sign in with Google" button on login page
  • Handle callback in auth/callback route
Effort: Small — mostly config
