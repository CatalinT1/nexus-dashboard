🔴 Post-Commit Bug Fix (Applied Today)

INSERT missing organization_id — would have broken all record creation

The multi-tenancy migration added organization_id NOT NULL to clients, staff, and appointments tables, and RLS WITH CHECK policies enforce the org match on INSERT. However, the Day 4 commit did not update the INSERT payloads in the app — any attempt to create a new client, appointment, or staff member would have failed with a NOT NULL / RLS violation error.

Fixed by adding getOrgId() calls before each INSERT in four files:

// Before (all three INSERT sites — would throw) const { error } = await supabase.from("clients").insert({ name, email, ... }); // After (applied to clients/new, appointments create, inline client create, staff create) const orgId = await getOrgId().catch(() => null); if (!orgId) { setError("Could not determine organization."); return; } const { error } = await supabase.from("clients").insert({ name, email, ..., organization_id: orgId });
Verified clean: npx tsc --noEmit returned no errors after all four INSERT fixes were applied.
