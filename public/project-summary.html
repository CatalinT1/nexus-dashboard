📋 Project Overview

A production-quality SaaS management dashboard built from scratch, featuring real authentication, a live Supabase PostgreSQL database, full CRUD operations, a calendar, reports with charts, staff management, and a client CRM — all in a single Next.js 16 App Router project.

Local URL: http://localhost:3000  ·  Directory: d:\Dashboard\app

🗄 Database Schema (Supabase)

All tables have Row Level Security enabled. SELECT requires an authenticated session. Write operations (INSERT/UPDATE) are further scoped to admin or manager roles.

profiles

  • id → auth.users
  • name, email
  • role (admin/manager/staff)
  • phone, department
  • is_active, joined_at
  • avatar (nullable)

clients

  • id, name, email, phone
  • company (nullable)
  • status (active/vip/prospect/inactive)
  • address_street/city/state/zip/country
  • notes, tags (TEXT[])
  • total_appointments, total_revenue
  • last_contact_at

staff

  • id, name, email
  • role, specialization
  • is_active
  • work_days (INT[])
  • start_time, end_time
  • total_appointments
  • rating (NUMERIC 3,1)

appointments

  • id, client_id → clients
  • staff_id → staff
  • client_name, staff_name (denorm.)
  • title, category, status
  • start_time, end_time
  • fee, color, location
  • reminders (BOOLEAN)

client_activities

  • id, client_id → clients
  • type (appointment/note/email/call/payment)
  • title, description
  • created_by (TEXT)
  • created_at

notifications

  • id, user_id → auth.users (nullable)
  • type (info/success/warning/error)
  • title, message
  • is_read, href
  • created_at

revenue_data

  • id
  • month (TEXT), month_order (INT)
  • year (INT)
  • revenue (NUMERIC)
  • appointments (INT)
Auto-profile trigger: When a new user signs up in Supabase Auth, a PostgreSQL trigger (handle_new_user()) automatically creates a row in the profiles table using the auth user's ID, email, and optional name from raw_user_meta_data.

🔐 Authentication Flow

  1. 1
    Route Protection — proxy.ts Every request passes through the proxy. It calls supabase.auth.getUser() (validates JWT server-side). Unauthenticated requests to /dashboard, /clients, etc. are redirected to /login. Authenticated users visiting /login are redirected to /dashboard.
  2. 2
    Sign In — /login Calls supabase.auth.signInWithPassword({ email, password }). On success: router.refresh() then router.push("/dashboard"). On failure: displays the Supabase error message in a red alert.
  3. 3
    Dashboard Layout — (dashboard)/layout.tsx Async Server Component. Calls getUser() then queries the profiles table for name/role/avatar. Passes user data as a prop to the Header component.
  4. 4
    Password Reset /forgot-password calls resetPasswordForEmail() with a redirectTo pointing to /auth/callback?next=/reset-password. The callback route exchanges the code for a session, then the /reset-password page calls updateUser({ password }).
  5. 5
    Sign Out Header calls supabase.auth.signOut() then navigates to /login. The proxy blocks any back-navigation attempt.

⚠️ Important Notes

Supabase Redirect URL: To make password reset emails work on localhost, go to Supabase Dashboard → Authentication → URL Configuration and add http://localhost:3000/** to the Redirect URLs list.
RLS & Roles: All database tables use Row Level Security. Data is only visible to authenticated users. Write operations (add client, add staff) are restricted to users whose profiles.role is admin or manager. The seeded admin user has full access.

💡 Key Technical Decisions

DecisionWhy
proxy.ts instead of middleware.ts Next.js 16 renamed the middleware file and export. Using the old name silently fails.
getUser() not getSession() in proxy getSession() trusts the cookie without server-side JWT validation — a security risk. getUser() validates with Supabase on every request.
@supabase/ssr (not auth-helpers) The auth-helpers-nextjs package is deprecated. @supabase/ssr is the correct package for Next.js App Router.
Server Components for read-heavy pages Dashboard, client detail, and reports pages fetch data server-side with Promise.all() for parallel queries — no loading state needed, better SEO.
Recharts extracted to "use client" children Recharts is a client-only library. Server Component pages extract chart rendering into separate charts.tsx files marked "use client".
await cookies() in server.ts Next.js 16 requires cookies() to be awaited — calling it synchronously throws a build error.
await params in Server Components Next.js 16 makes params a Promise in Server Components — must await params before reading route segments like [id].
